Entry 01 · April 2026

Starting from nothing

This is the first proper documentation entry. The project technically started on April 21 while I was still figuring out what this project even is. As of April 23, I have a spec document submitted to KTH (voluntary feedback), an empty GitHub repo with the folder structure set up.

What went wrong: I initially gravitated toward Processing (Java) because it is course-standard, and I wrote a full spec for a different concept (“Liminal”). That spec is coherent on paper, but it did not survive contact with my actual constraints and interests. I changed direction, which means I currently have multiple partially-formed starting points: a strong rendering idea (SDF-only antagonist shadow), an incomplete story (characters and world exist, branching narrative does not), and technical unknowns I still need to resolve (grading equivalence for browser-based p5.js + GLSL, whether a three-room vignette is sufficient scope, and how to produce normal maps for 2D pixel art).

Academic anchor: the current technical framing is a graphics-first horror design where the rendering system is the primary expressive layer (dynamic lighting via \(N \cdot L\), SDF-based shadow mathematics, and post-processing artifacts), and the evaluation plan in the submitted spec is a small user study using the Self-Assessment Manikin (SAM) scale with \(n \ge 5\).
